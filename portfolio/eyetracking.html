Eye Tracking

Know Where People Look During Usability Testing

What Is It?

Eye Tracking is my intern project at Epic Systems in the summer of 2014. It is an application with 3 parts:


1. it lets designers create eyetracking usability tests (record where participants look)

2. conducts them to collect data

3. analyzes the collected data via visualizations.


The purpose of this project is to be an affordable alternative to the $30,000 Tobii suite. We used another device called The Eye Tribe which only costs $100.

How Is It Made?

We started off with only The Eye Tribe's API reference, and built up the rest of the application from scratch. We used C# for the bulk of our application, WPF for the frontend, and SQL for data storage. We used Visual Studio and SQL Management Studio.

What Did I Do?

I worked with one other person on this project. We were given a large control, so we got to decide on exactly what to make. To achieve this, first I extensively researched the applications, strengths, and weaknesses of eyetracking studies. I designed and wrote the backbone data structures, and created APIs to access them. gaze replay gaze plot sql data access layer conduct usability test design GUI
